# MoA 웹사이트 구축 명세서 (Blueprint)

## 1. 프로젝트 개요
- **프로젝트명**: MoA (Subscription Sharing Platform)
- **타겟 사용자**: 2030 세대 (합리적인 소비, 트렌드 민감)
- **목표**: 구독 서비스(OTT 등)를 안전하고 저렴하게 공유하는 중개 플랫폼 구축
- **컨셉**: "혼자 내면 부담, 같이 내면 반값" - 친근하고 트렌디한 B2C 웹사이트

## 2. 디자인 및 UI/UX 가이드라인
- **톤앤매너**:
  - **컬러**: 신뢰감 있는 Indigo/Blue를 메인으로, 활기찬 Rose/Purple을 포인트로 사용.
  - **타이포그래피**: Pretendard 또는 Noto Sans KR (가독성 및 현대적인 느낌).
  - **레이아웃**: 딱딱한 관리자 대시보드 형태를 탈피하고, 소비자 친화적인 웹사이트 구조 (상단 GNB + 전체 너비 콘텐츠) 적용.
  - **모션**: 부드러운 호버 효과, Blob 애니메이션, 플로팅 요소로 생동감 부여.
- **언어**: 한국어 (딱딱한 명사형 종결을 피하고 친근한 '해요'체 사용).

## 3. 페이지별 상세 명세

### 3.1 메인 홈 (Landing Page)
- **히어로 섹션**:
  - 감성적인 그라디언트 배경 및 Blob 애니메이션 적용.
  - 메인 카피: "혼자 내면 부담, 같이 내면 반값"
  - 주요 CTA: [파티 찾기], [파티 만들기]
  - 비주얼: 플로팅 카드로 실제 절약 가능한 금액을 직관적으로 시각화.
- **특장점 섹션 (Features)**:
  - 핵심 가치 전달: 먹튀 방지(보증금), 즉시 매칭, 자동 정산.
  - 직관적인 아이콘과 설명 배치.
- **인기 파티 미리보기**:
  - 현재 모집 중인 인기 파티를 카드 형태로 노출하여 유입 유도.

### 3.2 파티 찾기 (Party List)
- **기능**:
  - 검색: 서비스명(넷플릭스, 유튜브 등)으로 검색.
  - 필터: 카테고리(영상, 음악, 생산성) 탭 필터링.
- **UI 요소**:
  - 그리드 레이아웃의 카드 디자인.
  - 서비스 아이콘, 파티장 닉네임, 월 분담금 강조.
  - 모집 인원 현황을 프로그레스 바(Progress Bar)로 시각화.

### 3.3 마이페이지 (My Page)
- **개인화 대시보드**:
  - 유저 프로필 영역 (우수 회원 뱃지 등).
  - 통계 요약: 총 절약 금액(누적), 다음 결제일 D-Day, 참여 중인 파티 수.
  - **절약 차트**: D3.js를 활용하여 일반 구독 대비 MoA 이용 시 절약 금액 비교 그래프 제공.
- **나의 구독 목록**:
  - 현재 이용 중인 파티 리스트.
  - 상태값(이용중, 일시정지, 문제발생 등)에 따른 시각적 구분.

### 3.4 AI 어시스턴트 (MoA 봇)
- **기능**: Google Gemini API 연동 챗봇.
- **페르소나**: 2030 세대에게 맞는 친절하고 트렌디한 말투 사용.
- **UI**: 우측 하단 플로팅 버튼 (FAB) 및 오버레이 채팅창.

## 4. 데이터 모델 (간략)
- **User**: 이메일, 닉네임, 프로필 이미지, 역할(User)
- **Party**: 서비스명, 제목, 설명, 월 분담금, 최대 인원, 현재 인원, 상태(모집중/마감/진행중)
- **Subscription**: 서비스 종류(Netflix, Disney+ 등), 요금제 정보

## 5. 기술 스택
- **Frontend**: React 18, TypeScript
- **Styling**: Tailwind CSS
- **Visualization**: D3.js
- **AI**: Google Gemini API (@google/genai)
- **Routing**: React Router v6+
